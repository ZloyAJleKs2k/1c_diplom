# Диплом по 1С для Курса 


ИНДИВИДУАЛЬНОЕ ЗАДАНИЕ

Вид программы:	Профессиональная переподготовка
Название программы:	Программирование на языке 1С в АПК
Фамилия Имя Отчество	…

Тема: «Создание конфигурации для работы предприятия,
осуществляющего торговлю кормами для животных»

В конфигурации должны быть учтены основные учетные функции предприятия. В том числе в обязательном порядке должны быть реализованы:
1.	Поступление товаров. (в документах должен быть реализован функционал подстановки, расчета нескольких реквизитов при заполнении). 
2.	Продажа товаров. (в документах должен быть реализован функционал подстановки, расчета нескольких реквизитов при заполнении).
3.	Учет сведений о товарах, в т.ч. о сроке годности, ценах закупки, ценах продажи, поставщиках. (учет информации о ценах закупки оптимально реализовать в разрезе поставщиков с использованием регистра сведений).
4.	Учет остатков товаров на складе.
5.	Формирование отчетов об остатках товаров, о сроках годности, о ценах закупки / продажи товаров. 
Описанный функционал является минимальным, при его реализации необходимо создать в системе все необходимые и вспомогательные объекты. С учетом полученных в ходе курса знаний рекомендовано расширить возможности системы, добавив свою индивидуальную логику.

В конфигурации должны быть использованы следующие объекты системы:
● Подсистемы;
● Справочники;
● Документы;
● Журналы документов (по необходимости);
● Регистры сведений;
● Регистры накопления;
● Отчеты.
В программном коде следует использовать следующие конструкции:
● Процедуры;
● Функции;
● Циклы;
● Условия;
● Запросы.
Процедуры и функции сформированы в:
● Модулях объектов;
● Модулях форм;
● Общих модулях.

Использованы клиентские и серверные процедуры (функции) с директивами
компиляции:
● &НаКлиенте;
● &НаСервере.

При выполнении аттестационной работы необходимо:
1. Добавить и настроить необходимые справочники, документы, журналы;
2. Добавить и настроить необходимые регистры накопления, сведений;
3. Настроить процесс передачи данных в регистры;
4. Реализовать механизмы, при помощи которых система будет производить
расчеты и формировать отчеты;
5. Добавить подсистемы и распределить по ним созданные объекты конфигурации.
